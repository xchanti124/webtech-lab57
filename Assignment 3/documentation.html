Media API

This is a CRUD API to manage the media database.

Get all media

URI

GET -> /media

Response

On success, the response is a list of all the media available in the database. It is an array of objects, where each object represents a single piece of media. It has the following data shape: 

{
  id: number;
  name: string;
  year: string;
  genre: string;
  poster: string;
  description: string;
}

A status code of 200 is returned on success.

On error, the response is an object with a single field error: 

{ error: string; }

Create new media

URI

POST -> /media

Request

The request should be a valid JSON representing a new media. The Content-Type of the request is expected to be application/json. The body of the request should follow the following shape: 

{
  name: string;
  year: string;
  genre: string;
  poster: string;
  description: string;
}

Response

On operation success, the ID of the newest entry is returned:

{ id: number; }

A status code of 200 is returned on success.

On error, the response is an object with a single field error: 

{ error: string; }

A status code of 400 is returned when invalid JSON or invalid arguments are provided.

Read media

URI

GET -> /media/:id

Request parameters

  • :id - ID of the media to get

Response

On success, the response is an object representing a single piece of media. It has the following data shape: 

{
  id: number;
  name: string;
  year: string;
  genre: string;
  poster: string;
  description: string;
}

A status code of 200 is returned on success.

On error, the response is an object with a single field error: 

{ error: string; }

A status code of 404 is returned when invalid ID is provided.

Update media

URI

PUT -> /media/:id

Request parameters

  • :id - ID of the media to update

Request body

The request body should provide fields that should be updated. The accepted fields are: name, year, genre, poster, description. All are optional. When provided, each field should be a string. The full data shape looks as follows: 

{
  name?: string;
  year?: string;
  genre?: string;
  poster?: string;
  description?: string;
}

Response

On operation success, a success message in the following shape is returned: 

{ message: string; }

A status code of 200 is returned on success.

On error, the response is an object with a single field error: 

{ error: string; }

A status code of 400 is returned when invalid JSON or invalid arguments are provided.
A status code of 404 is returned when invalid ID is provided.

URI

DELETE -> /media/:id

Request parameters

  • :id - ID of the media to delete

Response

On operation success, a success message in the following shape is returned: 

{ message: string; }

A status code of 200 is returned on success.

On error, the response is an object with a single field error: 

{ error: string; }

A status code of 404 is returned when invalid ID is provided.
